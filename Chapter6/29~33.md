# 비옥한 토양
- 프로젝트와 팀은 더 큰 조직이라는 맥락 안에 존재 -> 해당 맥락을 기업 문화
  - 어떤 문화는 건강한 업무 환경을 제공
  - 어떤 문화는 업무 자체가 불가능하게 만듬
- 보통 우리의 통제 범위 외 존재하지만 고려할 만한 가치 존재
  - 최악의 경우 위에서 무엇을 강요하는지 알아야함

# 자체 치유 시스템
##### 확정적 시스템과 비확정적 시스템
- 확정적 시스템
  - 명시적인 계획한 응답만 내놓음
  - 자체 교정 능력이 없음
  - 통제 불가능한 자체 교정 능력을 없애버린다는게 장점일 수도 있음
    - 처음부터 완벽하게 설계해야하지만 모든 엣지 케이스에 대해 한번에 고려하여 반영하기는 힘듬
- 비확정적 시스템
  - 우아하고 원만하게 스스로 교정 가능
    - 시스템을 구성하는 사람들이 근본적인 목표를 잘 이해할때 가능 -> 새로운 상황에 대처 가능
- 조직은 어떤면에서 시스템이며, 현실에서는 확정적 시스템을 만들자고 주장 -> 화자는 반대하는 입장
##### 절대 방법론에 숨겨진 의미
- 그저 그런 직원과 무능력한 직원으로도 우수한 조직을 만드는 방법 -> 절대 방법론
  - 고도의 사고가 요구되는 작업을 수행하는 일련의 방식으로 변형하는 시스템적 이론
  - 절대 방법론을 작성하는 사람들은 똑똑해야하지만 수행하는 사람들은 멍청해도 됨
- 절대 방법론을 따르면 자체 교정 능력을 잃어버림 -> 말도 안되는 방향으로 진행되는 일이 발행(부정적 효과)
- 절대 방법론과 그냥 방법론의 차이
  - 그냥 방법론: 업무를 완수하는 기본적인 접근 방식
  - 절대 방법론: 중앙에서 사고를 제어하려는 시도
    - 사람들이 스스로 사고할 만큼 충분히 똑똑하지 못하다는 전제를 깔고감
##### 절대 방법론의 광기
- 절대 방법론은 업무를 수행할 능력이 충분한 사람들에게 극심한 피해를 끼침 -> 틀에 맞게 강요하면 아래와 같은 역효과 발생
  - 산더미 같은 서류 업무
    - 사람들이 일보다 서류 작성에 시간을 더 쏟음 -> 병적인 문서 집착
  - 방법 부족
    - 좋은 방법이 수천 가지라면 하나를 골라 표준화 하려는 시도는 타당
    - 방법이 몇가지 정도인데, 하나를 골라 표준화 하면 다른 좋은 방법들은 배제하게 됨
  - 책임 부재
    - 잘못이 사람이 아닌 절대 방법론에게 묻게됨
  - 전반적인 사기 저하
    - "위에서 우리를 무능력하다고 여김"다는 인식이 듬
##### 악의적인 준수라는 문제
- 사람들이 절대 방법론을 무시하지 않고 정확히 따르는 문제
  - 시간 낭비하고, 돌아가지 않는 제품이 나오고, 무의미한 문서가 쌓임 -> 악의적 준수
##### 아기와 목욕물
- 절대 방법론으로 얻어지는 이익의 대다수는 방법 수렴으로 얻어지는 이익
  - 같은 일을 하는 사람들이 같은 방법을 같은 방식으로 사용 -> 빠르게 처리되고 익혀짐
- `절대 방법론이 방법을 수렴하는 유일한 방책은 아님` -> 방법 수렴하는 더 좋은 방법은 아래와 같음
  - 교육: 어떻게 하는지 잘 아는 일을 함
  - 도구: 자동화된 도구가 방법 수렴보다 효과적
  - 동료 검토: 동료 검토 활동 활발히 하면 자동적으로 수렴하게 됨
##### 첨단 기술 다시 생각하기
- 절대 방법론에 대한 조직의 집착은 최첨단 환상의 또다른 예
  - 기술이라는 믿음에서 나오는 집착

# 위험과 함께 춤을
- 소프트웨어 프로젝트에서의 리스크 관리 내 정반대인 행동 유형 두가지
  - 위험 관리 없는 위험 감수
  - 야심적인 성취를 원천 봉쇄해 버리는 위험 회피
- 위험 관리를 제대로 하지 않는다면 조직의 정치와 문화가 원인일 가능성이 높음
##### 위험에서 달아나지 않기
- 프로젝트에서 위험은 좋은 것 -> 가치가 있다는 증거
  - 가치가 있고 위험이 없는건 이미 옛날에 다 해치움
- 위험 있는 프로젝트를 성공 했을 때는 큰 보상을 얻음
- 위험 있는 프로젝트가 가지는 위험 요소들은 아주 많음
  - 위험 목록을 작성하라고 한다면 분명 하나를 빼먹을 것임 -> 우리 자신의 실패
##### 아마 우리가 결코 관리하지 않을 위험 한 가지
- 우리가 빼먹기 쉬운 위험은 우리 자신의 실패라는 위험
  - 아마 패배주의로 보일수도 있다고 생각해서 위험 요소로 안꼽을수도 있음
- 위험 관리 목적은 위험 발생 시 완화 목적
  - 위험을 없애기 위한 목적이 아님
  - 완화하려면 미리 잘 계획하고 준비해야함
##### 미이행(지연) 위험을 관리하지 않는 이유
- 결과가 도전으로 여겨지면 '할 수 있다'라는 사고가 위험 관리를 대체
  - 역경에 맞서 자신을 증명하고자 최선을 다하자는 마음
  - 일정이 중요할 수록 완화 계획에 쏟을 시간이 적어지고, 사람들도 안하고 싶어짐
- 관리자와 팀이 위험 관리르 않을 작정이라면 다른 누구라도 해야함
